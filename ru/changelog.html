                  # v2.5.12 (341)

                  • Добавлена поддержка разделения резервных копий данных на файлы размером 1 ГБ для обхода ограничения файловой системы FAT32
                  • Добавлена возможность разблокировать трекеры
                  • Добавлена опция пропуска проверки подписи при восстановлении резервных копий
                  • Поддержка Termux: run-as – отладка приложения или запуск нового сеанса в виде приложения во вкладке «О приложении».
                  • Отображание информации о резервной копии приложения на главной странице
                  • Восстановление исходных файлов (за исключением файлов APK) отключено на неподдерживаемых архитектурах
                  • Отображение диалогового окна подтверждения операции перед очисткой данных приложения
                  • Возможность импорта компонентов отключена с помощью IFW на MAT
                  • Включение внешнего носителя и каталога OBB для резервного копирования
                  • Разрешение импорта существующих правил другими приложениями или инструментами
                  • Добавлена опция для извлечения значков приложений во вкладке «О приложении»
                  • Отображание функции восстановления и удаления резервных копий только для приложений с существующими резервными копиями
                  • Отображание индикатора прогресса во время создания резервной копии
                  • Отображение индикатора прогресса во время загрузки операций приложения
                  • Исправлено неоткрытие приложения в последней и единственной поддерживаемой версии Aurora Droid (версия 1.0.6)
                  • Исправлен сбой приложения при смене ночного режима во время просмотра страницы «Сведения о приложении»
                  • Исправлен сбой приложения при попытке открыть внешний файл APK
                  • Исправлена ошибка NullPointerException, когда внешний каталог данных имел значение NULL
                  • Исправлено отображение панели инструментов в полноэкранном диалоге
                  • Исправлен поиск без учета регистра
                  • Оптимизация темы приложения
                  • Замена AndroidShell на LibSuperuser
                  • апрос разрешения доступа к внешнему хранилищу при сохранении файлов APK
                  • Обход ошибки AppBarLayout в Material Design
                  • Обновление внешней информации об APK при установке/удалении событий

                  Чтобы использовать возможности Termux, убедитесь, что вы используете Termux версии 0.96 или новее, а команда allow-external-apps=true добавлена в ~/.termux/termux.properties.

                  Функция резервного копирования данных по-прежнему является экспериментальной, и поэтому, пожалуйста, не полагайтесь на нее при управлении своими резервными копиями.

                  # v2.5.11 (333)

                  • Добавлена экспериментальная поддержка резервного копирования данных приложений. Пожалуйста, тестируйте только те приложения, которые вам не нужны. (только в режиме root)
                  • Добавлена возможность отправки разделенных файлов APK в формате APKS (можно установить через SAI).
                  • Реализовано сохранение файлов APK в режиме пакетного выбора.
                  • Добавлен список изменений для файла APK, который нуждается в обновлении (при открытии внешних файлов APK).
                  • Добавлена возможность применения операций в один клик к системным приложениям (по умолчанию отключено).
                  • Добавлена отображение информации о версии установленного приложения во вкладке «О приложении». Нажатие на значок i открывает вкладку «О приложении».
                  • Новые разрешения по запросу READ_EXTERNAL_STORAGE и WRITE_EXTERNAL_STORAGE для поддержки резервного копирования приложений
                  • Отображение удаленных приложений, у которых есть резервные копии, на главной странице
                  • Добавлен отказ от ответственности
                  • Исправлено неочищение выбора после выполнения задачи на главной странице
                  • Преобразование различной информации во вкладке конфигураций и функций в текст для улучшения читаемости.
                  • Исправлен сбой на главной странице при фильтрации приложений по поисковому запросу
                  • Исправлен сбой во вкладке «О приложении» когда наличие каталога внешних данных дает ложноположительный результат

                  Примечание: резервные копии данных хранятся в /sdcard/AppManager, а резервные копии файлов APK хранятся в /sdcard/AppManager/apks. Резервное копирование данных в настоящее время не работает на Android Lollipop.

                  # v2.5.6 (233)

                  • Пакетные операции на главной странице: очистка данных приложения, отключение запуска в фоновом режиме, отключение/закрытие/удаление приложения (нажмите на значок приложения для выбора)
                  • Полная поддержка экспортированных и поврежденных файлов с помощью Blocker.
                  • Повторная реализация блокирующих активити, приемников, служб и поставщиков
                  • Удалена зависимость ConstraintLayout, поэтому возможно уменьшение размера приложения
                  • Исправлено предупреждение о дублировании использования приложения на странице «О приложении».
                  • Исправлен сбой при ненахождении значка приложения на странице «Сведения о приложении».

                  Примечание для пользователей root: чтобы гарантировать, что предыдущие правила блокировки сохраняются с новой реализацией блокировки, это обновление основано на предыдущих правилах, следовательно, увеличивая время загрузки на главной странице. Эта функция будет удалена в следующей версии приложения, но ее все еще можно будет симулировать, применив глобальную блокировку компонентов в настройках приложения.
